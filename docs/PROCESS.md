# 運用ルール（MD更新とmain統合）

## MD更新の頻度
- `PROJECT.md`（低頻度）: マイルストーン完了やアーキ変更時のみ更新。通常はスプリント終了時、もしくは大きな設計変更が入ったときに追記。
- `TODO.md`（高頻度）: 作業開始/完了のたびにチェックボックス更新。新しいタスク着手時に追加し、完了後に `[x]` へ。
- その他ドキュメント: 新しい運用ルールは `docs/` 配下に追加し、変更時は TODO 更新で触れる。

## main への統合方針
1. 作業ブランチで小さめのPRを作成（タイトル例: `feat: homework list UX`）。  
2. マージ条件:
   - `npm run build` を通過（必要なら lint も）。
   - `TODO.md` に最新の着手/完了状況を反映。
   - 大きな仕様変更時のみ `PROJECT.md` を更新（低頻度ルール）。
3. レビュー: 可能なら1名レビュー。軽微修正はセルフマージ可だが、翌日にTODOへ反映。
4. マージ後: ブランチを削除し、次のタスクは新ブランチで開始。
